## INT8 rknn 파일로 odroid m1에서 C를 이용한 추론 실행 시 디텍션 수가 너무 많이 나오는 문제 발생
- 기존 FP16 c 추론 성공 main.cpp 파일 그대로 INT8 추론 실행했을 땐 디텍션이 아예 이루어지지 않았음
- 기존 클래스 스코어 찾는 좌표 float score = data[(4 + c) * num_anchors + i];로 구성되어 있었음
- 바운딩 박스 좌표 float cx = data[0 * num_anchors + i]; 로 구성되어 있었음

-> dequantize(역변환)을 이용하는 코드로 1차 수정하였으나 디텍션 이루어지지 않음

-> v::cvtColor(img, img, cv::COLOR_BGR2RGB); FP16은 BGR/RGB 바뀌어도 어느 정도 물체를 찾지만 INT8은 명확히 설정해주어야 해서 설정함

-> outputs[i[.want_float = 0 설정 및 scale, zp 값 불러와 수동 역양자화 공식 구현

-> num_anchor 변수를 이용하여 좌표 받아오는 형태로 수정하였는데 디텍션을 3000개 내외로 하는 상황 발생

-> 기존 NCHW 방식(channel * 8400 + i) 에서 NHWC(i * 5 + offset) 방식으로 수정하여 디텍션 수 100개 내외로 낮춤

-> 디버그 코드 추가하여 라벨 파일은 정상적으로 읽어지고 있는지, 지표 산출 데이터(TP, FP, FN) 값 확인하였음(TP:0, FP: 179941, 총 FN: 28166)
